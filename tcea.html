The things necessary to get the OpenVPN server running in Bridged Mode:

Note: Replace 192.168.0.1 with the internal IP of your OpenVPN server. Replace 1.1.1.1 with the external IP of your OpenVPN Server. Replace 10.16.0.1 with the other internal IP of your OpenVPN server (where 10.16.0.0 is an open subnet.) Clients will get IPs on the 10.16.0.1 subnet.

1. Two ethernet interfaces connected to the switch. eth0 has an IP of 192.168.0.1 (Mapped to external 1.1.1.1)
eth1 has an IP of 10.16.0.1.
2. The Main Switch has a static route for the 10.16.x.x subnet point to 192.168.0.1.
3. IP forwarding is enabled on the server (In /etc/sysctl.conf.) Make sure the line "net.ipv4.ip_forward=1" exists.
4. Correct OpenVPN configuration (at /etc/openvpn/server.conf.)
5. Push the routes to the clients for all the subnets. (Done in /etc/openvpn/server.conf)
6. Have UDP enabled through the Firewall on port 1194.
7. Have ethernet Bridge installed correctly (Done in /etc/network/interfaces.)
8. A line in /etc/rc.local to have the OpenVPN server start on reboot.
9. Certificates created with easy-rsa. (/etc/openvpn/easy-rsa/) See https://help.ubuntu.com/10.04/serverguide/C/openvpn.html.
10. Place up.sh and down.sh in /etc/openvpn/
